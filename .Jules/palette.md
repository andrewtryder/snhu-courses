## 2024-07-03 - Accessible Comboboxes & Focus States
**Learning:** For combobox dropdowns, `role="listbox"` strictly expects its direct children to be `role="option"`. Status items (like "Searching..." or "No matching courses") cannot use `role="status"` or `role="alert"` as siblings to options within the listbox. Instead, they must be `role="option"` with `aria-disabled="true"`. To properly announce updates, use a visually hidden `aria-live="polite"` region outside the combobox that updates with context (e.g. "3 suggestions available. Use up and down arrows to navigate."). Additionally, external links that open in new tabs should always use visually hidden text like "(opens in a new tab)" to warn screen reader users. Always use `focus-visible:ring-2` to support explicit keyboard navigation rather than relying on default browser styling.
**Action:** Apply `role="option" aria-disabled="true"` for non-interactive items in custom select/combobox elements. Utilize dedicated visually hidden `aria-live` containers for communicating complex dynamic state to screen readers. Add visually hidden "(opens in a new tab)" text to `target="_blank"` links. Apply explicit `focus-visible` styles to all interactive elements globally.
